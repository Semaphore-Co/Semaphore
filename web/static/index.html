Semaphore lets you express things sincerely

You can make groups, share questions with friends, give them the answer to let them make up questions, and more
If you have a question you don't want to ask directly, it's likely that it will fit in with the rest of the questions on Semaphore

Semaphore takes privacy seriously

both for the asker

  • All of your interactions are stored in a distributed format on the blockchain, so no one from server admins to hackers can see them
  • We ensure that you can't tell which of your friends use Semaphore, so even if you're the only one, there is no way for the answerer to know

and for the answerer

  • Users are always told that the question could have been asked by someone else; that's part of the fun
  • Before joining a group, users are always shown what kinds of things will become public by joining

Frequently Asked Questions

  • Why is it called Semaphore?
    • In programming, a process is often split up into "threads", so that it can be run at the same time on multiple CPU or GPU cores and be faster than it would be otherwise. A semaphore is a variable that is used to make sure that different threads don't try to access and change a common resource at the same time, which would result in the program having weird behavior, called concurrency problems. Our version of Semaphore essentially does the same thing: it controls access to information in a manner that ensures that no one gets hurt. You can read more about semaphores in programming on the Wikipedia page.
  • What is the graphic on the front page?
    • The blockchain is an extremely important aspect of Semaphore's technology, so we decided it would be apt to put a related graphic on the front page. In order for transaction participants to sign off that something actually happened, you need a digital signiture that no one else can replicate. The way that this is done is by having everyone keep a private key and a public key. Once you encrypt something with one, you need to have the other in order to decrypt it. That way, you can use the public key to ensure that someone that had the private key agreed to the transaction. The public/private key algorithm that is typically used relies on elliptic curve cryptography, the continuous version of which is illustrated by the graphic. You can read more about the specifics in a blog post from Cloudflare.
  • What currency do you use for transactions?
    • The distributed portion of Semaphore is built on the Etherium block chain, and uses a token as the currency for all transactions. As Semaphore grows, we expect demand for the token to rise along with the price, but we expect to keep the prices stable with respect to USD in order to give users a more consistant experience.
